# Sales Statistics Analyzer

מערכת דפדפן מקומית לניתוח סטטיסטי של לידים ומכירות מתוך קובץ CSV.

המערכת נבנתה כפרויקט עצמאי לגיטאהב: אין תלות במערכות CRM אחרות, אין שרת צד-אחורי, ואין צורך במסד נתונים.

## יכולות מרכזיות

- העלאת קובץ CSV של לידים.
- הפעלת ניתוח סטטיסטי על קובץ דוגמה אנונימי.
- מדדי KPI: סך לידים, מעוניינים, לא רלוונטיים, תהליך אחר, הצעות מחיר ויחס עניין.
- ניתוח סטטוס ליד, מקור הגעה, פוטנציאל, מנהל לקוח וסטטוס מכירה.
- סיבות סגירה מחושבות רק עבור לידים לא רלוונטיים.
- לידים ללא סיבת סגירה נספרים כתהליך אחר ולא כחסם.
- דוח מנהלים אוטומטי עם המלצות פעולה.
- ציון איכות לכל ליד.
- רשימת לידים לטיפול מיידי.
- יעדים מול ביצועים עם שינוי יעדים בזמן אמת.
- בקרת איכות נתונים: טלפונים, מקורות, מנהלים, סטטוסים, תאריכים וכפילויות.
- ניתוח איכות לפי פוטנציאל לקוח.
- תוכנית פעולה אוטומטית לפי היעדים והנתונים המסוננים.
- העתקת דוח מנהלים ללוח.
- ייצוא נפרד של לידים לטיפול מיידי.
- ניתוח לפי שעות פעילות וימי שבוע.
- מגמה יומית.
- משפך לידים.
- איכות לפי מקור הגעה.
- ביצועי מנהלי לקוח.
- מטריצת מקור הגעה מול סטטוס ליד.
- ניתוח אותות מתוך הערות: אין מענה, הצעת מחיר, פגישה, תקציב, טלפון שגוי ועוד.
- קובץ דוגמה פיקטיבי מורחב עם 100 לידים מכל סוגי הסטטוסים והמקורות.
- איתור טלפונים כפולים ולידים תקועים.
- ייצוא CSV מורחב של הסיכום וההמלצות.

## הפעלה מקומית

### Windows

לחץ פעמיים על:

```bat
start_windows.bat
```

הדפדפן ייפתח בכתובת:

```text
http://localhost:8787
```

### macOS / Linux

```sh
sh start_mac_linux.sh
```

ואז לפתוח:

```text
http://localhost:8787
```

### פתיחה ישירה

אפשר לפתוח את `index.html` ישירות בדפדפן. העלאת CSV ידנית תעבוד, אבל כפתור `הפעל ניתוח סטטיסטי` עשוי להיחסם בחלק מהדפדפנים בגלל מגבלות `file://`.

## מבנה CSV נתמך

המערכת מותאמת לעמודות הבאות:

```text
טלפון ראשי
נוצר בתאריך
שם העסק
מנהל לקוח
סטטוס לקוח
מקור הגעה
פוטנציאל לקוח
סטטוס ליד
סטטוס מכירה
האם נשלחה הצעת מחיר
סטטוס הצעת מחיר
סיבת סגירה
מידע נוסף
דואר אלקטרוני
```

## פרטיות מידע

התיקייה כוללת קובץ דוגמה אנונימי בלבד:

```text
data/sample-leads.csv
```

אין להעלות לגיטאהב קבצי לידים אמיתיים עם טלפונים, שמות עסקים, מיילים או הערות לקוח. `.gitignore` כולל חסימות בסיסיות לקבצים כאלה.

## פרסום ב-GitHub Pages

אפשר לפרסם את הפרויקט כאתר סטטי:

1. העלה את התיקייה לריפוזיטורי בגיטאהב.
2. פתח `Settings`.
3. עבור ל-`Pages`.
4. בחר `Deploy from a branch`.
5. בחר את הענף `main` ואת התיקייה `/root`.
6. שמור.
