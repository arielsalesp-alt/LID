# Sales Statistics Analyzer

מערכת עצמאית לניתוח סטטיסטי של לידים ומכירות מתוך קובץ CSV.

הפרויקט בנוי כיישום סטטי שמתאים גם ל-GitHub Pages וגם להרצה מקומית, בלי תלות בשרת אחורי או במערכת CRM אחרת.

## מה יש במערכת

- מסך מרכזי אחד עם KPI, תובנות אוטומטיות ומרכז בקרה
- כפתורי ניווט בין דוחות במקום גלילה ארוכה
- דוח סקירה מרכזית: סטטוסים, משפך, ציוני איכות, יעדים ותוכנית פעולה
- דוח מקורות וזמני הגעה: מקורות, מגמה יומית, שעות, ימי שבוע, מטריצת מקור מול סטטוס והמלצות שיווק
- דוח צוות: ביצועי מנהלי לקוח, עומס פתוח, לידים חמים, לידים תקועים ואיכות לפי פוטנציאל
- דוח איכות וסיכונים: סיבות סגירה, אותות מהערות, איכות דאטה, כפילויות וגיל לידים
- דוח תפעול ורשומות: לידים לטיפול מיידי, לידים תקועים, טבלת ביצועים ורשומות מסוננות
- ייצוא CSV של סיכום מלא ושל לידים לטיפול
- העתקת דוח מנהלים בלחיצה אחת

## קובץ דוגמה

המערכת כוללת קובץ דוגמה פיקטיבי בלבד:

```text
data/sample-leads.csv
```

הקובץ כולל 100 לידים פיקטיביים לצורך הדגמה בלבד.

## הרצה מקומית

### Windows

```bat
start_windows.bat
```

### macOS / Linux

```sh
sh start_mac_linux.sh
```

לאחר ההפעלה פתח:

```text
http://localhost:8787
```

## פרסום ב-GitHub Pages

1. מעלים את התיקייה לריפוזיטורי GitHub.
2. נכנסים ל-`Settings`.
3. פותחים את `Pages`.
4. בוחרים `Deploy from a branch`.
5. בוחרים את `main` ואת `/root`.

## קבצים חשובים

- `index.html` - גרסה עצמאית מלאה ל-GitHub Pages
- `assets/app.js` - לוגיקת הניתוח
- `assets/styles.css` - עיצוב המערכת
- `data/sample-leads.csv` - נתוני הדגמה פיקטיביים
- `docs/GITHUB_UPLOAD.md` - הוראות העלאה ל-GitHub

## פרטיות

המערכת מיועדת לעבוד עם נתוני דוגמה פיקטיביים בלבד בריפוזיטורי הציבורי. לא מעלים ל-GitHub קבצי לידים אמיתיים.
